AuthController

O AuthController gerencia autenticação de usuários via JWT (JSON Web Tokens) e autenticação básica. Inclui endpoints para login, registro, renovação de tokens, recuperação de senha, e middlewares para verificação de tokens.

Propriedades

cript_salt

Tipo: number

Qualificadores

  • static
  • private

Guarda o cryp salt value para uso na criptografia das senhas

userEmail

Tipo: string

Qualificadores

  • static

Guarda email do usuário usado para enviar emails pela api

userEmailPassword

Tipo: string

Qualificadores

  • static

Guarda a senha do usuário do email usado para enviar emails pela api

mailTransport

Tipo: any

Qualificadores

  • static
  • private

Usado para armazenar o objeto que controlra o envio de emails

login

Tipo: request handler method

Parâmetros

  • req: Request
  • res: Response
  • next?: NextFunction

Tipo retorno: Promise<void>

Qualificadores

  • static
  • async

Autentica um usuário e retorna um token de acesso e um refresh token.

Detalhes do Endpoint

  • Método: POST
  • URL: /api/controllers/auth/authcontroller/login

Propriedades esperadas em req.body

  • email: string - Email do usuário.
  • password: string - Senha do usuário.
  • currentTimeZoneOffset?: number - Offset do fuso horário.

Exemplo de Requisição


  POST /api/controllers/auth/authcontroller/login
  Content-Type: application/json

  {
    "email": "user@example.com",
    "password": "senha123",
    "currentTimeZoneOffset": 0
  }

Exemplos de respostas

Status: 200


  {
    "success": true,
    "message": "logged",
    "data": {
      "token": "jwt_token",
      "refreshToken": "refresh_token",
      "user": {
        "id": 1,
        "email": "user@example.com",
        "access_profile_id": 1,
        "last_token": "jwt_token",
        "last_timezone_offset": 0
      }
    },
    "exception": null
  }

Status: 401 (erro)


  {
    "success": false,
    "message": "user not found",
    "data": null,
    "exception": null
  }

check_token

Tipo: middleware method

Parâmetros

  • req: Request
  • res: Response
  • next: NextFunction

Tipo retorno: Promise<void>

Qualificadores

  • static
  • async

Middleware que verifica a validade do token JWT em rotas protegidas. Anexa o usuário autenticado ao req.user.

Detalhes do Endpoint

  • Método: GET
  • Header: x-access-token: jwt_token
  • Rotas Não Seguras: Ignora verificação para /api/online, /api/endpoints, endpoints de autenticação, e /api/controllers/modules/registers/midiascontroller/upload_file.

Propriedades esperadas

  • x-access-token: string - Token JWT.

Exemplo de Requisição


  GET /api/protected-route
  x-access-token: jwt_token

Exemplos de respostas

Status: 401 (erro)


  {
    "success": false,
    "message": "token not provided",
    "data": null,
    "exception": null
  }

Inicialização Estática

Configura os seguintes métodos como request handler atravéz do métido EndPointsController.markAsRequestHandler, que possibilita que recebam diretamente requisições:

  • login
  • basic_auth
  • refresh_token
  • register
  • check_token
  • send_email_recover_password
  • password_change
